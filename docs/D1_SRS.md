## **ระบบอ่านการ์ตูนออนไลน์ OwlBook**  **1\. บทนำ (Introduction)**

### **1.1 วัตถุประสงค์ (Purpose)**

เอกสาร SRS ฉบับนี้จัดทำขึ้นเพื่อระบุความต้องการของระบบ OwlBook ซึ่งเป็นเว็บแอปพลิเคชันสำหรับอ่านการ์ตูนออนไลน์ ประกอบด้วยการสมัคร/เข้าสู่ระบบ การอ่านการ์ตูน การจัดซื้อและปลดล็อกตอนด้วยเหรียญ ตลอดจนการจัดการโดยผู้ดูแลระบบ เพื่อให้ทีมพัฒนานำไปออกแบบและพัฒนาได้อย่างถูกต้องตรงตามความต้องการ

### **1.2 ขอบเขตของระบบ (Scope)**

* ผู้อ่านการ์ตูนทั่วไปสามารถค้นหา อ่าน และจัดเก็บรายการโปรดได้  
* รองรับการอ่านการ์ตูนรายตอนแบบฟรีและระบบซื้อด้วยเหรียญ  
* ผู้ใช้สามารถสมัครสมาชิก เติมเหรียญ ดูประวัติธุรกรรม  
* ผู้ดูแลระบบสามารถเพิ่ม/แก้ไข/ลบข้อมูลการ์ตูนและตอนต่าง ๆ  
* รองรับการใช้งานผ่าน PC และ Mobile

### **1.3 คำจำกัดความ**

| คำศัพท์ | ความหมาย |
| :---- | :---- |
| User | ผู้ใช้งานทั่วไป |
| Admin | ผู้ดูแลระบบ |
| Coin | เหรียญในระบบใช้ซื้อ / ปลดล็อกตอน |
| Episode | ตอนของการ์ตูนแต่ละเรื่อง |

### **1.4 ภาพรวมของเอกสาร (Overview)**

\- **ส่วนที่ 1 บทนำ** ระบุวัตถุประสงค์ ขอบเขตของระบบ OwlBook คำนิยามศัพท์เทคนิค และโครงสร้างของเอกสาร  
**\- ส่วนที่ 2 คำอธิบายโดยรวม** อธิบายภาพรวมของระบบในมุมกว้าง ฟังก์ชันหลักที่ระบบทำได้ รวมถึงข้อจำกัดและสภาพแวดล้อมในการทำงาน  
**\- ส่วนที่ 3 ความต้องการเฉพาะ** รายละเอียดเชิงลึกของความต้องการเชิงหน้าที่ (Functional Requirements) เช่น การจัดการสมาชิก ระบบเหรียญ และความต้องการด้านคุณภาพ (Non-functional Requirements) เช่น ความปลอดภัยและความเร็ว  
**\- ส่วนที่ 4 อินเทอร์เฟซภายนอก** อธิบายส่วนติดต่อผู้ใช้ (UI) และการเชื่อมต่อกับซอฟต์แวร์หรือฮาร์ดแวร์อื่น ๆ  
**\- ส่วนที่ 5 ฟีเจอร์ระบบ** รายละเอียดการทำงานเชิงเทคนิคของฟีเจอร์สำคัญในระบบ  
**\- ส่วนที่ 6 ภาคผนวก** ข้อมูลเพิ่มเติมที่เกี่ยวข้อง เช่น User Stories หรือแผนผังต่าง ๆ เพื่อเสริมความเข้าใจ

## **2\. คำอธิบายโดยรวม (Overall Description)**

### **2.1 มุมมองระบบ (Product Perspective)**

ระบบเป็นเว็บแอปอ่านการ์ตูนออนไลน์ ใช้ฐานข้อมูลเก็บข้อมูลผู้ใช้ การ์ตูน เหรียญ และธุรกรรม โดยมุ่งเน้น UX ที่เรียบง่าย โหลดไว และเหมาะกับมือถือ

### **2.2 ฟังก์ชันหลักของระบบ (Major Functions)**

| หมวดฟีเจอร์ | รายละเอียด |
| :---- | :---- |
| ระบบสมาชิก | สมัคร/เข้าสู่ระบบ/ออกจากระบบ |
| ค้นหาและแสดงการ์ตูน | หน้า Main Page แสดงรายการการ์ตูน ค้นหา และกรองตามหมวดหมู่ |
| อ่านการ์ตูน | ดูรายละเอียดตอน อ่านแบบเลื่อนหน้า บันทึกความคืบหน้า |
| ระบบเหรียญ | เติมเหรียญ ซื้อปลดล็อกตอน ประวัติรายการ |
| รายการที่ถูกใจ | เพิ่ม/ลบ Favorite List |
| Admin | CRUD การ์ตูนและตอน |
| Mobile Responsive | รองรับการใช้งานบนมือถือ |

### **2.3 ข้อจำกัดของระบบ (Constraints)**

* ตอนเสียเงินต้องตรวจสอบสถานะสมาชิกก่อนอ่าน  
* ผู้ใช้ต้องมีเหรียญเพียงพอสำหรับการปลดล็อก  
* การแสดงผลต้อง Responsive สำหรับมือถือ  
* ระบบต้องโหลดตอนภายในเวลาที่เหมาะสม

## **4\. ความต้องการอินเทอร์เฟซภายนอก (External Interface Requirements)**

### **4.1 User Interface (UI)**

* หน้า Register / Login  
* หน้า Main Page แสดงรายการการ์ตูน \+ Search \+ Filter  
* หน้า Comic Detail (ชื่อเรื่อง รายละเอียด ตอนทั้งหมด)  
* หน้า Reader (เลื่อนอ่าน, next/previous)  
* หน้าเติมเหรียญ / ประวัติรายการ  
* หน้า Favorite  
* หน้า Admin Dashboard

### **4.2 Hardware Interface**

* ใช้งานผ่าน Browser บนอุปกรณ์ใดก็ได้ที่รองรับ HTML5

### **4.3 Software Interface**

* Web browser: Chrome/Firefox/Safari  
* Database: SQL/NoSQL ตามเลือกใช้  
* Payment API: (กรณีระบบเติมเหรียญภายหลัง)

## **5\. ฟีเจอร์ระบบ (System Features – Detailed)**

### **5.1 Member System**

* Email Validation, Password ≥ 8  
* Session / JWT Login  
* Logout clears session instantly

### **5.2 Comic Browsing**

* Infinite scroll หรือ pagination  
* Search by keyword  
* Category filter

### **5.3 Reading**

* Reader UI แบบ Scroll  
* Episode status: Free/Locked  
* Auto-save progress

### **5.4 Coin & Purchase**

* Coin Wallet  
* Purchase Episode  
* History Transaction tracking

### **5.5 Favorite Feature**

* Quick Access list

### **5.6 Admin Panel**

* Manage Comic Master Data  
* Upload episode / pages
